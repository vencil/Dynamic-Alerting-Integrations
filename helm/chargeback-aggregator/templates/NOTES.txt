Hard rule (#552): the gateway access-log request count cannot bill — a
heavy and a light query cost the same number of requests. This pipe-
line reads from `prometheus_query_log` (where `samples_scanned` /
`exec_time_s` actually live) and aggregates **offline**, so a slow
chargeback run never wedges the federation hot path.

Rows with empty `tenant_id` are bucketed as `platform` — these come
from Prometheus's own rule / alert evaluations (no tenant_id selector).
Useful for sanity-checking the platform's own load, NOT billable.
