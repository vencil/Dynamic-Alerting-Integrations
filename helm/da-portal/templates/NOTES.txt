NetworkPolicy only works if your CNI ENFORCES it. Flannel (base) and some managed
  "simple" CNIs SILENTLY IGNORE NetworkPolicy: the API server accepts it and this
  install succeeds, but traffic is left unrestricted (security theater). Use a
  NetworkPolicy-aware CNI (Calico / Cilium / Antrea) and verify by sending a packet
  that SHOULD be denied — do not trust a successful install. The CNI-independent
  root-cause fix is L7 identity verification (roadmap #5); this policy is L4
  defense-in-depth.
