## Bug-fix — `add_frontmatter.py` ✅ 已修復

### 症狀（事後追記）

```bash
python scripts/tools/dx/add_frontmatter.py --check --base-dir docs/
```

**預期**: 只報哪些檔案缺 frontmatter，不動檔案。
**實際**: 真的寫檔，而且寫進 FUSE 上的 symlink 代理（把 `docs/CHANGELOG.md` 15 bytes 的 `"../CHANGELOG.md"` 覆蓋成 112 bytes 的 `---\ntitle: ...\n---\n\n../CHANGELOG.md`）。

發現當下已手動復原：`Write("docs/CHANGELOG.md", "../CHANGELOG.md")`。repo-root 的 `CHANGELOG.md` 未受影響（Python 寫的是 FUSE 代理檔，沒跟著 symlink 追到實體檔）。

### 根因

- `main()` line 346 呼叫 `process_file(..., dry_run=args.dry_run)`——`args.check` 沒有 OR 進去
- `find_markdown_files()` 不檢查 `os.path.islink`
- 不走 `doc_coverage.py` 的 `EXCLUDE_*` 清單
- 順手發現一個副作用 bug：scan_dirs 同時包含 `base_dir/docs` 和 `base_dir`，`os.walk` 會重複訪問同一個檔案兩次（「Found 2 markdown files」代表 1 個真實檔案）

### 本 session 的修復（已落地）

1. `main()` 新增 `read_only = args.dry_run or args.check`，呼叫改成 `dry_run=read_only`——`--check` 永遠不寫
2. `find_markdown_files()`：
   - 新增 `if os.path.islink(filepath): continue`（與 `doc_coverage.py` 對齊，skip 3 個 symlink 代理）
   - 新增 `_is_excluded()` helper 比對 `EXCLUDE_RELATIVE_PATHS` / `EXCLUDE_PATH_PREFIXES` / `EXCLUDE_PATH_GLOBS`
   - 收尾用 `sorted(set(os.path.realpath(p) for p in md_files))` dedup
3. 檔案頂端新增 `EXCLUDE_*` 常數（與 `doc_coverage.py` 同步）
4. 移除 `if args.dry_run: ... else: ...` 的死 branch（兩條都 `logger.info(message)`）

## ⚠️ Gotcha — doc-map 的 `--check` 呼叫姿勢

`.pre-commit-config.yaml` 裡 `doc-map-check` hook 實際執行：

```
python3 scripts/tools/dx/generate_doc_map.py --check --include-adr
```

**不要**只跑裸 `--check`——會因為 default `include_adr=False` 把現有含 ADR 的
`doc-map.md` 誤判為「extra entries」。本地要手動驗證時也必須帶 `--include-adr`。

同理，重新產生時也要帶：

```bash
python scripts/tools/dx/generate_doc_map.py --generate --lang all --include-adr
```
