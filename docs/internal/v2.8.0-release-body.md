## 📋 v2.8.0 — Customer migration pipeline + 1000-tenant scale validation + automation consolidation

**Highlights:**

- 🚚 **Customer migration automation** — 4 new Go CLIs (`da-parser` / `da-tools profile build` / `da-batchpr` / `da-guard`) onboard a customer's existing `PrometheusRule` corpus into this platform's `conf.d/` Profile-as-Directory architecture; **1000-tenant onboarding now takes ~1 day instead of ~1 week**. Three delivery paths (Docker / static binary 6-arch / air-gapped tar) ship with every release tag, each cosign-signed + SBOM SPDX/CycloneDX. See [Migration Toolkit Installation](https://github.com/vencil/Dynamic-Alerting-Integrations/blob/v2.8.0/docs/migration-toolkit-installation.md).
- 📊 **Tenant Manager at 1000+ tenant scale** — UI now calls `/api/v1/tenants/search` (server-side search + pagination, page_size cap 500, p99 < 200 ms @ 1000 tenants) instead of fetching the full DEMO list. Smart Views frontend wires to the v2.5.0 backend `/api/v1/views` CRUD; TenantCard footer adds Alert Builder / Routing Trace deep-link buttons.
- 🔬 **Defaults change is previewable** — `POST /api/v1/tenants/simulate` ephemeral endpoint lets CI and UI dry-run the effect of `_defaults.yaml` edits on inheritance **before** commit (no disk IO, no manager state mutation). The new `da_config_blast_radius_tenants_affected` histogram quantifies the per-tick affected-tenant distribution.
- 🔒 **Q2 2026 CVE consolidation** — Grafana / Prometheus 3.x / Alertmanager / oauth2-proxy / alpine-git / Python base / Go toolchain bumped together, clearing **50+ CVEs (13 CRITICAL, 35+ HIGH)** covering auth bypass, RCE, memory safety, and TLS DoS. See umbrella issue [#100](https://github.com/vencil/Dynamic-Alerting-Integrations/issues/100).
- 🛡️ **API boundary hardening** — `tenant-api` now fails-fast on out-of-range patches (key/value over-length, `_timeout_ms` > 1 hour, `_silent_mode` non-enum → 400 + full violations array). Flat + nested mixed-mode duplicate tenant id promoted from WARN to typed `*DuplicateTenantError` hard error (state preservation invariant).

### Benchmark (1000 tenants, Dev Container, Intel Core 7 240H, Go 1.26.2)

| Metric | Time | Meaning |
|:---|---:|:---|
| `FullDirLoad_1000` | 112 ms | Cold load (scan + YAML parse + merge + hash) |
| `IncrementalLoad_1000_NoChange` | 1.3 ms | Dual-hash steady-state reload (86× faster than cold) |
| `Simulate_DeepChain` | 5 ms | `/simulate` endpoint per-call |
| 5-anchor e2e fire-through p99 | < 2 s | conf.d write → webhook receiver (n=30, bootstrap 95% CI) |

SLO maintained from v2.7.0: cold load 112 ms @ 1000 tenants; steady-state reload 1.3 ms is 0.0087% of the default 15-second scan interval — effectively zero overhead. Full report: [`docs/benchmarks.md`](https://github.com/vencil/Dynamic-Alerting-Integrations/blob/v2.8.0/docs/benchmarks.md).

### ⚠️ Breaking changes

- **Mixed-mode duplicate tenant id** — Same tenant id appearing in both flat (`tenants/<id>.yaml`) and nested (`conf.d/.../<id>.yaml`) layouts now rejects load with typed `*DuplicateTenantError` (previously WARN + last-wins). Run `da-tools validate-config --strict` before upgrading to catch silently-tolerated duplicates.
- **`tenant-api` body validation** — Out-of-range patches return 400 + full violations array instead of being silently accepted. CI pipelines with previously-tolerated bad payloads will fail immediately after upgrade.
- **Open-mode `PUT/DELETE Groups`** — In open-mode environments (no `_rbac.yaml`), `PUT/DELETE Groups` now returns 403 because tenant-scoped authz writes are denied by default. Fix: add minimal `_rbac.yaml`:
  ```yaml
  groups:
    - name: dev
      tenants: ["*"]
      permissions: [admin]
  ```
