# Contract-test (schemathesis) pinned dependency — SSOT for CI, Makefile
# and README. Install with: pip install -r tests/contract/requirements.txt
#
# Why pinned (the repo otherwise installs CI Python tools unpinned):
# run_contract_tests.py depends on schemathesis 4.x CLI semantics
# (`--url`, `--max-examples`, `--checks <name,...>`, and the
# `--suppress-health-check filter_too_much` used to absorb the
# seed-dependent generation-efficiency flake on constraint-heavy ops).
# An unpinned `pip install schemathesis` tracked *latest* on every CI
# run, so a new release could silently drift the flake rate, rename a
# health check, or break a flag — the "local green, CI first-run red"
# version-drift class. Freezing the version makes the contract-fuzz
# behavior reproducible and pushes every upgrade through an explicit,
# reviewable PR. See issue #1158.
schemathesis==4.23.0
#
# hypothesis is pinned EXPLICITLY, not left to schemathesis's transitive
# range (`hypothesis<7,>=6.130.12`). This is the load-bearing pin for the
# flake: `filter_too_much` is a *hypothesis* HealthCheck, and hypothesis
# owns the data-generation / health-check behavior that drives the seed
# flake. Pinning schemathesis alone would still let hypothesis float to
# latest at install time — the actual flake source would drift. Freeze
# both together; Renovate / an explicit PR bumps them in lockstep.
hypothesis==6.156.9
#
# Upgrade trigger: bump deliberately (and re-verify the health-check /
# flag names still hold) when schemathesis crosses a major version
# (5.x), when hypothesis crosses a major (7.x), or when the
# filter_too_much flake recurs on these pinned versions.
